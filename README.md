# cat_vasya_game
Учебный проект по ООП: архитектура персонажей и заклинаний.


# Модуль 1
**Что добавлено:** Фундаментальный класс `Unit`, от которого наследуются все игровые сущности.
**Как реализовано:** 
  - Использованы `ABC` и декоратор `@abstractmethod` из модуля `abc`.
  - В конструкторе `__init__` сохранены 6 характеристик: `strength`, `dexterity`, `constitution`, `wisdom`, `intelligence`, `charisma`.
  - Объявлены три абстрактных метода: `calculate_max_health`, `calculate_damage`, `calculate_defense`. Реализация перенесена в наследников, чтобы Python запрещал создание экземпляров напрямую `Unit()`.

# Модуль 2
**Что добавлено:** Класс `Character`, реализующий всю расчётную логику внутри одного файла, без создания отдельных классов для воина/мага/охотника.
**Как реализовано:**
  - В конструктор добавлен параметр `character_class` с проверкой на вхождение в допустимые значения (`warrior`, `mage`, `hunter`). При ошибке выбрасывается `ValueError`.
  - Методы `calculate_*` содержат условные конструкции `if/elif`, выбирающие формулу в зависимости от `self.character_class`.
  - Все промежуточные результаты округляются вниз через `int()`, как указано в ТЗ.
  - В конце конструктора сразу вычисляются и фиксируются итоговые поля: `self.max_health`, `self.current_health`, `self.damage`, `self.defense`.

# Модуль 3
**Что добавлено:** Абстрактный класс `Spell`, три конкретных заклинания, пул маны и методы управления магией.
**Как реализовано:**
  - В `Unit` добавлены атрибуты `spells` (пустой список) и `mana` (0 по умолчанию), а также методы `add_spell(spell)` и `cast_spell(index)`. Последний проверяет корректность индекса и баланс маны, вычитает стоимость и возвращает результат `spell.cast()`.
  - В `Character` реализован `calculate_max_mana()` с формулами для каждого класса. При создании персонажа `self.mana` сразу приравнивается к максимальному значению.
  - Создан файл `spells.py` с классами `Fireball`, `IceLance`, `LightningBolt`. Они наследуются от `Spell`, принимают `name`, `damage`, `mana_cost` в конструкторе, а их метод `cast()` возвращает фиксированный урон.


## Структура
- `unit.py` — Абстрактный класс `Unit` с характеристиками, маной и методами управления заклинаниями.
- `character.py` — Класс `Character` с ветвлением формул урона, защиты и маны по `character_class`.
- `spells.py` — Абстрактный `Spell` и три конкретных заклинания (`Fireball`, `IceLance`, `LightningBolt`).
- `main.py` — Скрипт для демонстрации работы программы.
